Il s'agit d'un challenge disponible l'adresse : https://challengedata.ens.fr/participants/challenges/67/ propos par l'ENS.

L'objectif est de prvoir les "surge" dans deux villes ctires d'Europe occidentale.
Nous nous plaons dans une configuration de prvision : connaissant les valeurs de surge et le champ de pression au niveau de la mer des 5 derniers jours, nous voulons prdire les valeurs de surge des 5 prochains jours. Il s'agit donc d'un problme de prdiction de sries temporelles.

def surge_prediction_metric(y_true, y_pred):
    """
    Mtrique utilise pour mesurer la qualit de la prvision. 
    Les scores de prdictions pour les deux villes sont calcules indpendamment, et la perte finale est leur somme.
    Prend des dataframes en entre.
    """
    w = np.linspace(1, 0.1, 10)[np.newaxis]
    surge1_cols = [
        'surge1_t0', 'surge1_t1', 'surge1_t2', 'surge1_t3', 'surge1_t4',
        'surge1_t5', 'surge1_t6', 'surge1_t7', 'surge1_t8', 'surge1_t9' ]
    surge2_cols = [
        'surge2_t0', 'surge2_t1', 'surge2_t2', 'surge2_t3', 'surge2_t4',
        'surge2_t5', 'surge2_t6', 'surge2_t7', 'surge2_t8', 'surge2_t9' ]
    surge1_score = (w * (y_true[surge1_cols].values - y_pred[surge1_cols].values)**2).mean()
    surge2_score = (w * (y_true[surge2_cols].values - y_pred[surge2_cols].values)**2).mean()

    return surge1_score + surge2_score

Les valeurs de surge tant normalises, $1-$ peut tre vu comme un pourcentage de variance explique. Avec une prdiction zro triviale de toutes les valeurs, le score est $1$, ce qui signifie que nous expliquons 0 % de la variance. Un score suprieur 1 est donc pire que la prdiction nulle et peut tre considr comme mauvais.

Description des données¶

L'entre X est code au format numpy .npz et se compose de :

  • id_squence : les ids de la squence
  • t_slp : les 40 heures GMT auxquelles les champs de pression au niveau de la mer (SLP) sont donns.
  • slp : les 40 champs de pression au niveau de la mer (SLP), encods dans des images de taille (41, 41).
  • t_surge1_input : les 10 heures GMT auxquelles sont donnes les hauteurs de surge dans la ville 1.
  • surge1_input : les hauteurs de surge donnes dans la ville 1.
  • t_surge2_input : les 10 heures GMT auxquelles les hauteurs de surge sont donnes dans la ville 2.
  • surge2_input : les hauteurs de surge donnes dans la ville 2.
  • t_surge1_output : les 10 heures GMT auxquelles nous devons prdire les hauteurs de surge dans la ville 1.
  • t_surge2_output : les 10 heures GMT auxquelles nous devons prvoir les hauteurs de surge dans la ville 2.
